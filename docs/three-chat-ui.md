## 多面板
支持3种选项开启/关闭(需要在options页的通用设置增加 "总结面板设置" 的单独页面, 并把原来通用设置里和面板相关的都放到这里):
- 页面内嵌侧边栏面板 (顶部是model和prompt的选择框, 中间是对话历史, 底部是 开始总结/重新总结, 对话输入框和发送对话(同浮动面板一样,可以开关隐藏))
- 页面内浮动面板(默认开启, 和原有的类似)
- popup面板+固定(简单, 仅model/prompt选择和总结)



## 复杂度
页面内(content)的比较复杂, 需要支持右键菜单启动
需要支持 面板打开即可开启summary


## popup
popup 本身, 有一个layout, 就是和原来的一样, 显示插件名, version. 但是需要简洁, 新的layout应该是类似 win/linux/macos应用的窗口栏一样, 只有一行, 左边是内容,  右上角是跳转到设置和固定按钮
当面板开启popup选项后, 才显示简单的总结面板(默认固定)

## 结构
- 面板前端组件页面不复用

## 组件依赖
不再像原有的那样的手动简单实现, 而是使用 ai-elements 组件
